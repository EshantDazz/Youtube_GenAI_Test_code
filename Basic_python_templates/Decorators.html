Python Decorators

Advanced Function Modification Techniques

What are Decorators?

A decorator is a callable Python object (usually a function) that takes another function as its argument and returns a new function. It allows you to modify the behavior of a function without changing its source code. Think of it as wrapping a function with another function to add extra functionality.

Why Use Decorators?

Decorators offer several advantages:

  • Reusability: You can create a decorator once and apply it to multiple functions.
  • Separation of concerns: Decorators help keep your code clean by separating core functionality from additional features.
  • Readability: The @ syntax makes code more concise and easier to understand.
  • Common use cases:
    • Logging
    • Caching
    • Timing function execution
    • Authentication
    • Error handling
    • Profiling
Example Breakdown
def say_hello(func):
    def wrapper():
        print("Hello!")
        func()
        print("Goodbye!")
    return wrapper

@say_hello
def greet():
    print("How are you?")

# Using the decorator
greet()
  1. Defining the decorator:
    • say_hello is the decorator function. It takes another function (func) as an argument.
    • wrapper is the inner function that will be returned. It prints "Hello!", calls the original function (func), and then prints "Goodbye!".
  2. Applying the decorator:
    • @say_hello before greet means that greet is passed as an argument to say_hello.
    • The return value of say_hello (which is wrapper) is assigned to greet.
  3. Using the decorated function:
    • Calling greet() actually calls the wrapper function, which prints "Hello!", calls the original greet (printing "How are you?"), and then prints "Goodbye!".
Equivalent Without Using the Decorator Syntax
def greet():
    print("How are you?")

def say_hello(func):
    def wrapper():
        print("Hello!")
        func()
        print("Goodbye!")
    return wrapper

decorated_greet = say_hello(greet)
decorated_greet()

This code achieves the same result as the previous example, but it's less concise and readable. The decorator syntax provides a cleaner way to apply the decorator.

Key Points
  • Decorators are functions that take functions as arguments and return functions.
  • They are used to add functionality to existing functions without modifying their code.
  • The @ syntax is a convenient way to apply decorators.
  • Common use cases include logging, caching, timing, authentication, error handling, and profiling.

By understanding decorators, you can write more efficient, reusable, and maintainable Python code.
